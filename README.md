# Aphelion Systems Website

A static business website for a web, app, and digital systems studio. It is built with plain HTML, CSS, and JavaScript so it can run locally or deploy directly to GitHub Pages.

## Files

- `index.html` - concise homepage
- `services.html` - service and systems details
- `pricing.html` - suggested starting prices
- `about.html` - studio positioning and human presence
- `faq.html` - common client questions
- `process.html` - project process and standards
- `contact.html` - project inquiry form
- `thank-you.html` - FormSubmit redirect page
- `privacy.html` and `terms.html` - basic business policies
- `sitemap.xml` and `robots.txt` - search indexing helpers
- `styles.css` - responsive visual design
- `script.js` - mobile nav, theme toggle, and form status behavior
- `assets/hero-workspace.png` - generated hero artwork

## Preview

Open `index.html` in a browser, or serve this folder with any static file server.

## Launch checklist

- Use `aphelionsystems@gmail.com` until a paid domain and matching email are actually owned and configured.
- Submit one test form so FormSubmit confirms the Gmail destination.
- Update page titles, descriptions, OpenGraph text, and `sitemap.xml` if the GitHub Pages URL changes.
- Do not add a CNAME until a real domain has been purchased.
- Add Google Search Console by verifying the GitHub Pages URL or future custom domain.
- Analytics are not installed yet. Add a real Google Analytics, Plausible, or Cloudflare Web Analytics ID only after choosing an account.
- Review pricing, care plan details, privacy policy, terms, and contact fields before sending the site to prospects.
- Keep case studies current as more client work is completed.
